The Universal 90-Second Opening (Memorize This)

The first 90 seconds of any system design interview are the most important. Use this exact script as a skeleton — only the nouns change.

Seconds 0–20 — Frame requirements.
"Before I dive in, let me make sure I understand the scope. The functional asks are [X, Y, Z]. The non-functional dimensions I'll explicitly call out are offline-first, battery, and a p95 latency target. Are there any constraints on platform versions, regions, or scale I should know?"

Seconds 20–40 — Estimate scale.
"Let me derive a rough scale. Assuming ~N DAU × M sessions × K events, that's roughly X RPS peak and Y MB per user of local storage. That fits comfortably in Room and means a single OkHttp client with HTTP/2 connection pool."

Seconds 40–60 — Pick architecture, justify in one sentence.
"I'll use a single-source-of-truth Room database with a unidirectional flow: Repository → ViewModel (StateFlow) → Compose UI. Network responses write into Room; UI only ever reads from Room. This gives us offline support, predictable behavior across rotation and process death, and a clean test seam."

Seconds 60–80 — Component walkthrough.
"On the wire, OkHttp + Retrofit with an auth interceptor and refresh Authenticator; real-time via WebSocket wrapped in callbackFlow with exponential backoff; background work via WorkManager with NetworkType.CONNECTED constraint and idempotency keys."

Seconds 80–90 — Pre-empt failures.
"Before going deeper, let me name the failure modes I want to address: token expiry mid-request, WebSocket drops, process death mid-write, and Room migration failures. I'll cover each as I walk through the design."

If you say only that, you've already separated yourself from 70% of candidates. Now you can actually start designing.

Universal Cross-Questions (Asked of Every Design)

These come up in every mobile system design — be ready with one-line answers regardless of the system you're discussing.

"How do you handle process death?"
SavedStateHandle for in-flight UI state; Room/DataStore for durable data; WorkManager for queued work that must resume; on relaunch, restore from local before any network call.
"How do you handle offline?"
Room is the source of truth; UI observes a Flow from Room; writes go to Room first then an outbox; sync engine drains outbox via WorkManager when network returns.
"How do you handle authentication?"
OkHttp Interceptor adds the access token to every request; OkHttp Authenticator reactively refreshes on 401 and retries; refresh token in EncryptedSharedPreferences (Keystore-backed).
"How do you minimize battery impact?"
WorkManager with NetworkType.UNMETERED + BatteryNotLow constraints for non-urgent work; batch network calls; avoid wake locks; coalesce timers; respect Doze.
"How do you handle config changes (rotation)?"
ViewModel survives; stateIn(WhileSubscribed(5000)) keeps upstream alive across the brief detach; SavedStateHandle for primitive UI state.
"How do you test this?"
Repository as the seam: fake data sources implement the same interface; ViewModel tests via runTest + Turbine; UI tests via Compose-test with fake repositories.
"How do you instrument observability?"
Structured analytics events (see #10), traceparent header for distributed tracing, Crashlytics for stability, Firebase Performance for cold start & network latency, custom span markers via systrace.
"How does this scale to 100M users?"
Mobile-side: this is constant per device. Backend: stateless API tier, partition by user_id, CDN for static assets, push-based fanout has issues with celebrities, fall back to pull or hybrid.
"What about iOS parity?"
Architecture is platform-agnostic above the local-storage layer; share API contracts via OpenAPI; share business rules via KMP if the team adopts it. Don't share UI.
"Security — encryption at rest?"
Sensitive blobs via EncryptedSharedPreferences / EncryptedFile; Room SQLite via SQLCipher if needed; secrets in Android Keystore (hardware-backed where TEE available).

Anti-patterns That Tank Interviews

These are the moves I've personally watched candidates make that immediately drop them a level. Avoid all of them.

  • Treating the phone as a thin client. Saying "the client just calls a REST endpoint" instead of leveraging Room, WorkManager, Keystore.
  • Calling the network from the UI/ViewModel directly. Always: UI ← Room ← Repository ← Network.
  • Saying "I'll use MVVM" without explaining state ownership. Name who owns state, who mutates it, and how it survives rotation/process death.
  • Ignoring battery. No interview about a background-active feature is complete without WorkManager constraints.
  • Using runBlocking or GlobalScope. Both are red flags. Always lifecycleScope / viewModelScope / SupervisorJob with explicit dispatcher.
  • Using floats/doubles for money. BigDecimal or integer cents always.
  • Skipping idempotency. Every "create" or "purchase" needs a client-side UUID; server treats it as the primary key.
  • "I'll add caching later." Caching is a design decision, not an optimization. Specify it upfront with TTL and eviction policy.
  • Forgetting Process Death. "What if the OS kills my app right here?" is the question that separates Senior from Staff.
  • No mention of failure modes. Token expiry, WebSocket drops, partial batch failures, Room migration — name them before being asked.
  • Hand-waving conflict resolution. "We'll merge somehow" → "Server timestamp wins; loser preserved as a conflict copy" or "CRDT for collaborative".
  • Ignoring observability. Mentioning Crashlytics/traces/metrics signals production maturity.

1. Notes / TODO App with Sync

30-sec elevator pitch: "Room is the source of truth. The UI observes a Flow from Room, so writes feel instant. Each write also lands in an outbox table; a WorkManager job drains the outbox to the backend with idempotent client-side IDs. Conflict policy is server-version-wins with the loser kept as a conflict copy. SavedStateHandle preserves the in-progress draft across process death."
Junior says"I'll save the note to a server with a POST. If it fails I'll show a snackbar."
Staff says"Writes are local-first and idempotent. The outbox guarantees delivery across process death; WorkManager respects battery and network constraints; conflicts are explicit and resolvable."
Back-of-envelope: 10M DAU × ~20 notes/day × ~1 KB = ~200 GB/day write. Per device: ~10k notes × 1 KB = 10 MB local; trivial. Sync RPS at 10M DAU and 4 syncs/day = ~460 RPS — single backend tier.

1 What's the problem?

Build an Android app where a user can create, edit and delete notes, see them instantly even with no internet, and have them transparently sync to a backend and other devices when connectivity returns. Sounds tiny — but the interviewer is testing whether you understand offline-first, conflict resolution, and process death.

2 How should I think — questions to ask
  • Single-user (just sync between their own devices) or multi-user/shared notes?
  • Does a note have rich content (images, attachments) or just plain text + title?
  • What's the conflict policy: last-writer-wins, merge, or user-prompt?
  • How long do we keep deleted notes (tombstones)? 30 days?
  • Do we need full-text search? Pinning? Reminders/Notifications?
  • Auth: email/password, Google, anonymous?
  • Scale: ~10k notes per user max, or unbounded?
3 Starting the discussion (HLD & LLD)

HLD

Three layers: UI (Compose) → Repository → Local Room DB (source of truth). A SyncEngine (WorkManager) reads an outbox table and pushes pending changes to the backend; a periodic pull-sync (or push from FCM) brings down changes from other devices. The UI never reads from the network directly.

LLD

  • Note(id: String, title, body, updatedAt, deleted, dirty, version)
  • NoteDao exposes Flow<List<Note>>
  • NoteRepository writes locally with dirty=1; never blocks UI on network
  • SyncWorker uploads dirty rows, downloads changes?since=lastSyncAt, applies merges
  • Conflict policy: server updatedAt wins; loser kept as a "conflict copy" if needed
4 How to code & explain to attract

Open with: "My UI will only ever observe Room. Writes go to Room first with a dirty flag, then a WorkManager job pushes them. This makes the UI feel instant and survives process death because WorkManager persists its queue." Then sketch NoteDao, NoteRepository, SyncWorker in that order — data moves upstream of the UI, never the other way. Mention SavedStateHandle for the editor draft so a kill mid-typing doesn't lose work.

5 Actual code (Kotlin)
@Entity(tableName = "notes")
data class Note(
    @PrimaryKey val id: String,
    val title: String,
    val body: String,
    val updatedAt: Long,
    val deleted: Boolean = false,
    val dirty: Boolean = true,        // needs upload
    val version: Long = 0            // server version for OCC
)

@Dao interface NoteDao {
    @Query("SELECT * FROM notes WHERE deleted=0 ORDER BY updatedAt DESC")
    fun observe(): Flow<List<Note>>

    @Query("SELECT * FROM notes WHERE dirty=1")
    suspend fun dirty(): List<Note>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun upsert(n: Note)
}

class NoteRepository(private val dao: NoteDao, private val wm: WorkManager) {
    fun notes() = dao.observe()
    suspend fun save(n: Note) {
        dao.upsert(n.copy(updatedAt = System.currentTimeMillis(), dirty = true))
        wm.enqueueUniqueWork(
            "sync", ExistingWorkPolicy.KEEP,
            OneTimeWorkRequestBuilder<SyncWorker>()
                .setConstraints(Constraints.Builder()
                    .setRequiredNetworkType(NetworkType.CONNECTED).build())
                .build()
        )
    }
}

class SyncWorker(ctx: Context, params: WorkerParameters)
    : CoroutineWorker(ctx, params) {
    override suspend fun doWork(): Result = runCatching {
        val dirty = dao.dirty()
        val ack = api.push(dirty)               // idempotency: send id+version
        ack.accepted.forEach { dao.upsert(it.copy(dirty = false)) }
        val changes = api.pull(prefs.lastSyncAt)
        changes.forEach { remote ->
            val local = dao.findById(remote.id)
            if (local == null || remote.version > local.version)
                dao.upsert(remote.copy(dirty = false))
        }
        prefs.lastSyncAt = System.currentTimeMillis()
        Result.success()
    }.getOrElse { Result.retry() }                 // exponential backoff
}
6 Cross-questions & answers
Q. What if the user edits the same note on two devices offline?
A. Each write carries a server version. On push, the server compares versions; if mismatch → 409 Conflict. Client resolves by either (a) last-writer-wins on updatedAt, or (b) keeps both as "conflict copy". Talk about CRDTs (Yjs/Automerge) as the gold standard for collaborative editing.
Q. The user kills the app while typing — what happens?
A. Two safeguards: (1) SavedStateHandle persists the in-progress draft across process death; (2) we upsert the note on every debounced 500ms, so Room always has the latest committed version.
Q. How do you handle delete?
A. Soft delete with deleted=true + dirty=true. The server learns about the deletion on next push. We GC tombstones after 30 days so they don't grow forever.
Q. Why WorkManager and not just a coroutine?
A. WorkManager survives process death, respects Doze and battery saver, and lets us declare a NetworkType.CONNECTED constraint so we only attempt sync when there's network. A coroutine in viewModelScope dies with the screen.
Q. Same note opened on two tabs of the app — race?
A. Both observe the same Flow from Room; Room emits the latest after each upsert; UIs converge automatically. We debounce writes (300 ms) so we don't create write storms while typing.
Q. How big can a note get? Memory implications?
A. Cap at, say, 1 MB body. Stream large attachments separately; the note row only stores a content-addressed hash that points to a blob (see #15).
Q. Multi-tenant — sharing notes with another user?
A. Now we're in collaborative territory: switch conflict policy to CRDT (Automerge); push changes via WebSocket; permissions stored on the note, enforced server-side, mirrored client-side for UX.
Q. Backup / export?
A. Allow JSON export via ACTION_CREATE_DOCUMENT Storage Access Framework intent. For full restore, prompt the user to confirm overwriting local with imported content.
Q. What if Room schema changes between app versions?
A. Write explicit Migration objects, test with MigrationTestHelper, never use fallbackToDestructiveMigration() in prod, and keep schemas/ JSON exports under version control.
7 What can be better / alternatives
  • Real-time push: replace pull-sync with FCM "data" message + WebSocket for instant updates between user's devices.
  • CRDTs (Automerge, Yjs) for true collaborative editing without conflict copies.
  • End-to-end encryption: encrypt note body with a key derived from the user's password (Argon2 + Keystore) so the server can't read content.
  • Full-text search: Room FTS4 / FTS5 virtual table.
  • Attachments: chunked resumable upload to S3-style object storage with content-addressable hashes (see #15).
Don't say: "I'll just save it to a remote API and show a loading spinner." That's a Junior answer. Local-first is non-negotiable for a notes app.

2. Weather App (API + Caching)

30-sec elevator pitch: "Two caches, one source of truth. The OkHttp HTTP cache handles byte-level freshness via Cache-Control; the typed Room cache holds the structured forecast for the UI. Repository emits cached data instantly, then refreshes if stale. Locations from FusedLocationProvider with IP-fallback when permission denied. Widget refreshed via PeriodicWorkRequest."
Junior says"I'll call the weather API on screen open."
Staff says"Two-tier cache, server-driven Cache-Control TTLs, ETag-based 304s, fall back to last-known on offline with a stale indicator, never blank."
Back-of-envelope: 5 M DAU × ~6 opens/day = 30 M req/day. With 5-min app cache + ETag, hit ratio ~85% → ~52 RPS to weather provider. Per-user storage: 5 cities × 1 KB = 5 KB.

1 What's the problem?

Build an app that shows the current weather and a 7-day forecast for the user's location, refreshing it intelligently and being usable on a flaky network. The hidden lessons: cache invalidation, location permission handling, and HTTP cache vs app-level cache.

2 How should I think — questions to ask
  • How fresh must data be? Current = 5 min, forecast = 1 hour?
  • One location or multiple saved cities?
  • Use device GPS or IP-based location?
  • Background refresh allowed (e.g., for a home-screen widget)?
  • Units: metric/imperial, language localization?
  • Offline behavior: show last-known with a stale indicator?
3 Starting the discussion (HLD & LLD)

HLD

Compose UI → WeatherViewModel → WeatherRepository that consults (1) Memory cache → (2) Room cache → (3) Network. A FusedLocationProvider gives lat/long. A PeriodicWorkRequest updates a saved city in the background for the home-screen widget.

LLD

  • WeatherCache(lat, lon, current, forecast, fetchedAt)
  • Repo strategy: if cache age < TTL → emit cached; else fetch + store + emit
  • Use NetworkBoundResource-style flow that emits cache then network
  • OkHttp Cache as the second layer (HTTP-level caching)
4 How to code & explain to attract

Make it clear there are two caches: the OkHttp HTTP cache (cheap, byte-level, honors Cache-Control), and the app-level typed Room cache (structured, queryable, used as source-of-truth for the UI). Mention that you'd respect Cache-Control: max-age from the server when present, so the backend can change TTL without an app update. Always emit the cached value first so the UI never blanks; then refresh.

5 Actual code (Kotlin)
class WeatherRepository(
    private val api: WeatherApi,
    private val dao: WeatherDao,
    private val ttlMs: Long = TimeUnit.MINUTES.toMillis(5)
) {
    fun weather(lat: Double, lon: Double): Flow<Resource<Weather>> = flow {
        val cached = dao.get(lat, lon)
        cached?.let { emit(Resource.Loading(it.toDomain())) }

        val stale = cached == null ||
            System.currentTimeMillis() - cached.fetchedAt > ttlMs

        if (stale) {
            runCatching { api.fetch(lat, lon) }
                .onSuccess { fresh ->
                    dao.upsert(fresh.toEntity(System.currentTimeMillis()))
                    emit(Resource.Success(fresh))
                }
                .onFailure { e ->
                    if (cached != null)
                        emit(Resource.Stale(cached.toDomain(), e))
                    else
                        emit(Resource.Error(e))
                }
        } else emit(Resource.Success(cached!!.toDomain()))
    }.flowOn(Dispatchers.IO)
}

6 Cross-questions & answers
Q. The user is in a tunnel. What do you show?
A. Last cached forecast with a non-blocking "Last updated 12 min ago" banner. We never wipe cache on failure.
Q. How do you avoid hammering the API?
A. (1) App-level TTL, (2) OkHttp cache + ETag/If-None-Match for 304s, (3) debounce on user-initiated location changes, (4) cancel the in-flight request when the city changes.
Q. How does the home-screen widget update?
A. PeriodicWorkRequest every ~30 min with NetworkType.CONNECTED & BatteryNotLow constraints. Worker writes to Room; widget observes via AppWidgetProvider's onUpdate.
Q. Location permission denied — graceful path?
A. Fall back to IP-based geolocation (ipapi) or let the user pick a city manually. Save it in DataStore so we don't ask twice.
7 What can be better / alternatives
  • Use WorkManager + ExpeditedWork for "tap-to-refresh" widget updates.
  • Switch app-cache to StaleWhileRevalidate: emit cache instantly and refresh in background — reduces perceived latency to zero.
  • Push from server via FCM for severe weather alerts.
  • Server returns Cache-Control: max-age=N, stale-while-revalidate=M — client honors both.
Don't say: "I'll just refetch every time the screen opens." That's a Junior answer; you'll burn quota and battery and look bad on flaky networks.

Build a gallery that scrolls 100k+ device photos at 60fps, opens a full-screen pager with pinch-to-zoom, and supports basic actions (share, delete). The hidden curriculum: MediaStore queries, thumbnail decoding, memory pressure, and scoped storage.

2 How should I think — questions to ask
  • Scope: only DCIM & Pictures, or all media?
  • Group by date / album / location?
  • Do we need video too, or images only?
  • Edit operations (rotate, crop) needed?
  • Min API level — scoped storage matters from API 29+
  • Cloud sync (Google Photos style) in scope or out?
3 Starting the discussion (HLD & LLD)

HLD

MediaStore via ContentResolver is the source of truth. A Paging 3 PagingSource queries it in pages of 100; an image loader (Coil/Glide) decodes thumbnails sized exactly to the grid cell. A ContentObserver invalidates pages when new photos arrive. Full-screen viewer uses HorizontalPager with SubsamplingScaleImageView for huge bitmaps.

LLD

  • MediaItem(uri, dateTaken, width, height, mimeType, size)
  • MediaPagingSource.load(params) → cursor query with LIMIT/OFFSET
  • Thumbnail cache: LRU memory (1/8 heap) + DiskLruCache for decoded thumbs
  • Full-screen: load original via region decoder for > 4k images
4 How to code & explain to attract

The signal here is knowing how images blow up memory. A 12 MP photo decoded at full size = ~50 MB in ARGB_8888. For a 3-column grid you only need ~360px decoded. Tell the interviewer: "I always pass target dimensions to the loader so it downsamples at decode time, never after." Mention BitmapFactory.Options.inSampleSize as the underlying mechanism.

5 Actual code (Kotlin)
class MediaPagingSource(private val resolver: ContentResolver) :
    PagingSource<Int, MediaItem>() {

sealed class Token {
    data class Num(val v: BigDecimal) : Token()
    data class Op(val sym: Char, val prec: Int, val rightAssoc: Boolean = false) : Token()
    object LParen : Token()
    object RParen : Token()
}

fun tokenize(s: String): List<Token> = buildList {
    var i = 0
    while (i < s.length) {
        val c = s[i]
        when {
            c.isDigit() || c == '.' -> {
                val sb = StringBuilder()
                while (i < s.length && (s[i].isDigit() || s[i] == '.')) sb.append(s[i++])
                add(Token.Num(BigDecimal(sb.toString())))
            }
            c == '+' -> { add(Token.Op('+', 1)); i++ }
            c == '-' -> { add(Token.Op('-', 1)); i++ }
            c == '*' -> { add(Token.Op('*', 2)); i++ }
            c == '/' -> { add(Token.Op('/', 2)); i++ }
            c == '(' -> { add(Token.LParen); i++ }
            c == ')' -> { add(Token.RParen); i++ }
            c.isWhitespace() -> i++
            else -> throw IllegalArgumentException("bad char $c")
        }
    }
}

fun toRpn(tokens: List<Token>): List<Token> {
    val out = mutableListOf<Token>()
    val stack = ArrayDeque<Token>()
    tokens.forEach { t -> when (t) {
        is Token.Num -> out += t
        is Token.Op -> {
            while (stack.lastOrNull() is Token.Op &&
                   (stack.last() as Token.Op).prec >= t.prec) out += stack.removeLast()
            stack.addLast(t)
        }
        Token.LParen -> stack.addLast(t)
        Token.RParen -> {
            while (stack.last() != Token.LParen) out += stack.removeLast()
            stack.removeLast()
        }
    }}
    while (stack.isNotEmpty()) out += stack.removeLast()
    return out
}

fun eval(rpn: List<Token>): BigDecimal {
    val s = ArrayDeque<BigDecimal>()
    rpn.forEach { t -> when (t) {
        is Token.Num -> s.addLast(t.v)
        is Token.Op -> { val b = s.removeLast(); val a = s.removeLast()
            s.addLast(when (t.sym) {
                '+' -> a + b; '-' -> a - b; '*' -> a * b
                '/' -> a.divide(b, 10, RoundingMode.HALF_UP)
                else -> error("?")
            })
        }
        else -> {}
    }}
    return s.single()
}

data class Key(val url: String, val w: Int, val h: Int,
                val transforms: List<String>) {
    val hash: String by lazy { sha256("$url|$w|$h|${transforms.joinToString()}") }
}

class MemoryCache(maxBytes: Int) {
    private val lru = object : LruCache<Key, Bitmap>(maxBytes) {
        override fun sizeOf(k: Key, v: Bitmap) = v.byteCount
    }
    fun get(k: Key) = lru[k]
    fun put(k: Key, b: Bitmap) { lru.put(k, b) }
}

class ImageEngine(
    private val mem: MemoryCache,
    private val disk: DiskLruCache,
    private val http: OkHttpClient,
    private val pool: BitmapPool
) {
    private val inFlight = ConcurrentHashMap<Key, Deferred<Bitmap>>()
    private val scope = CoroutineScope(SupervisorJob() + Dispatchers.IO)

    fun load(req: Request): Job = req.lifecycle.coroutineScope.launch {
        val key = Key(req.url, req.w, req.h, req.transforms.map { it.id })

        mem.get(key)?.let { req.target.set(it); return@launch }

        req.target.set(req.placeholder)
        val bmp = inFlight.getOrPut(key) {
            scope.async { fetchAndDecode(req, key) }
        }.await()

        mem.put(key, bmp)
        req.target.set(bmp)
        inFlight.remove(key)
    }

    private suspend fun fetchAndDecode(req: Request, key: Key): Bitmap {
        disk.get(key.hash)?.use { snap ->
            return decode(snap.getInputStream(0).readBytes(), req)
        }
        val bytes = http.newCall(okhttp3.Request.Builder().url(req.url).build())
            .await().body!!.bytes()
        disk.edit(key.hash)?.let { e ->
            e.newOutputStream(0).use { it.write(bytes) }; e.commit()
        }
        return decode(bytes, req)
    }

    private fun decode(bytes: ByteArray, req: Request): Bitmap {
        val opts = BitmapFactory.Options().apply { inJustDecodeBounds = true }
        BitmapFactory.decodeByteArray(bytes, 0, bytes.size, opts)
        opts.inSampleSize = calcSampleSize(opts.outWidth, opts.outHeight, req.w, req.h)
        opts.inJustDecodeBounds = false
        opts.inMutable = true
        opts.inBitmap = pool.get(opts.outWidth / opts.inSampleSize,
                                 opts.outHeight / opts.inSampleSize)
        var bmp = BitmapFactory.decodeByteArray(bytes, 0, bytes.size, opts)
        req.transforms.forEach { bmp = it.apply(bmp, pool) }
        return bmp
    }
}

interface Loader<K, V> { suspend fun load(key: K): V }

interface Cache<K, V> {
    suspend fun get(key: K, loader: Loader<K, V>? = null): V?
    suspend fun put(key: K, value: V, ttlMs: Long = Long.MAX_VALUE)
    suspend fun evict(key: K)
}

data class Entry<V>(val value: V, val createdAt: Long, val ttlMs: Long) {
    fun expired() = System.currentTimeMillis() - createdAt > ttlMs
}

class TwoTierCache<K, V>(
    private val memMaxBytes: Int,
    private val disk: DiskCache<K, V>,
    private val sizer: (V) -> Int,
    private val defaultTtlMs: Long
) : Cache<K, V> {

    private val mem = object : LruCache<K, Entry<V>>(memMaxBytes) {
        override fun sizeOf(k: K, v: Entry<V>) = sizer(v.value)
    }
    private val inFlight = ConcurrentHashMap<K, Deferred<V>>()
    private val scope = CoroutineScope(SupervisorJob() + Dispatchers.IO)

    override suspend fun get(key: K, loader: Loader<K, V>?): V? {
        mem.get(key)?.takeIf { !it.expired() }?.let { return it.value }
        disk.get(key)?.takeIf { !it.expired() }?.let { e ->
            mem.put(key, e); return e.value
        }
        return loader?.let {
            inFlight.getOrPut(key) { scope.async { it.load(key) } }
                .await()
                .also { v -> put(key, v); inFlight.remove(key) }
        }
    }

    override suspend fun put(key: K, value: V, ttlMs: Long) {
        val jitter = (ttlMs * 0.1).toLong()
        val e = Entry(value, System.currentTimeMillis(),
                      ttlMs - Random.nextLong(jitter))
        mem.put(key, e); disk.put(key, e)
    }

    override suspend fun evict(key: K) { mem.remove(key); disk.remove(key) }
}

interface PagingSource<Key, Value> {
    suspend fun load(params: LoadParams<Key>): LoadResult<Key, Value>
}

sealed class LoadResult<Key, Value> {
    data class Page<Key, Value>(
        val data: List<Value>,
        val prevKey: Key?,
        val nextKey: Key?
    ) : LoadResult<Key, Value>()
    data class Error<Key, Value>(val e: Throwable) : LoadResult<Key, Value>()
}

class Pager<K, V>(
    private val source: PagingSource<K, V>,
    private val initialKey: K,
    private val pageSize: Int = 20,
    private val prefetchDistance: Int = 10
) {
    private val _items = MutableStateFlow<List<V>>(emptyList())
    val items: StateFlow<List<V>> = _items
    private val _state = MutableStateFlow<LoadState>(LoadState.Idle)
    val state: StateFlow<LoadState> = _state
    private var nextKey: K? = initialKey

    suspend fun loadNext() {
        val key = nextKey ?: return
        if (_state.value == LoadState.Loading) return
        _state.value = LoadState.Loading
        when (val r = source.load(LoadParams(key, pageSize))) {
            is LoadResult.Page -> {
                _items.value = _items.value + r.data
                nextKey = r.nextKey
                _state.value = LoadState.Idle
            }
            is LoadResult.Error -> _state.value = LoadState.Error(r.e)
        }
    }

    fun onPositionVisible(pos: Int, total: Int, scope: CoroutineScope) {
        if (total - pos <= prefetchDistance) scope.launch { loadNext() }
    }

    fun retry(scope: CoroutineScope) { scope.launch { loadNext() } }
    fun invalidate() {
        _items.value = emptyList(); nextKey = initialKey; _state.value = LoadState.Idle
    }
}

Q. Cursor vs offset — when do you pick which?
A. Cursor is correct for live feeds (insertions at head don't shift offsets); offset is fine for static datasets (catalogs). Cursor needs the server to expose a stable opaque token.

        onBufferOverflow = BufferOverflow.DROP_OLDEST)

    private lateinit var sessionId: String

            for (e in channel) dao.insert(e)        // persist immediately

    }

@Entity(tableName = "outbox")
data class OutboxEntry(
    @PrimaryKey val id: String = UUID.randomUUID().toString(),
    val op: String,           // CREATE | UPDATE | DELETE
    val entityType: String,
    val entityId: String,
    val payload: String,      // JSON
    val attempts: Int = 0,
    val createdAt: Long = System.currentTimeMillis()

class SyncEngine(
    private val db: AppDb,
    private val api: SyncApi,
    private val cursor: CursorStore,
    private val conflictResolver: ConflictResolver
) {
    suspend fun sync() = withContext(Dispatchers.IO) {
        push()
        pull()
    }

    private suspend fun push() {
        val batch = db.outbox().take(50)
        if (batch.isEmpty()) return
        val resp = api.push(batch.map { it.toDto() })
        db.withTransaction {
            resp.results.forEach { r ->
                when (r.status) {
                    "OK"       -> db.outbox().delete(r.id)
                    "CONFLICT" -> conflictResolver.resolve(r)
                    "RETRY"    -> db.outbox().incrementAttempts(r.id)
                    "FATAL"    -> db.outbox().markFailed(r.id, r.reason)
                }
            }
        }
    }

    private suspend fun pull() {
        val since = cursor.read()
        val changes = api.pull(since)
        db.withTransaction {
            changes.entities.forEach { applyMerge(it) }
            cursor.write(changes.serverNow)
        }
    }

    private fun applyMerge(remote: EntityDto) {
        val local = db.entities().findById(remote.id)
        if (local == null || remote.version > local.version) {
            db.entities().upsert(remote.toEntity())
        }
    }
}

    override suspend fun doWork() = try {
        engine.sync(); Result.success()
    } catch (e: IOException) { Result.retry() }

class UploadWorker(c: Context, p: WorkerParameters) : CoroutineWorker(c, p) {
    override suspend fun doWork(): Result {
        val id = inputData.getString("id") ?: return Result.failure()
        val job = dao.find(id) ?: return Result.failure()

        val session = job.sessionUrl ?: api.createSession(job.total).also {
            dao.update(job.copy(sessionUrl = it))
        }

        val chunkSize = 5L * 1024 * 1024
        val from = job.uploaded
        val to = (from + chunkSize).coerceAtMost(job.total)
        val chunk = readRange(applicationContext, Uri.parse(job.uri), from, to)

        val resp = api.uploadChunk(session, from, to - 1, job.total, chunk)
        if (!resp.isSuccess) return Result.retry()

        dao.update(job.copy(uploaded = to))
        setProgress(workDataOf("progress" to (to * 100 / job.total).toInt()))

        return if (to < job.total) Result.retry()             // schedule next chunk
        else {
            api.finalize(session)
            dao.update(job.copy(status = "DONE"))
            Result.success()
        }
    }
}

private fun readRange(ctx: Context, uri: Uri, from: Long, to: Long): ByteArray {
    ctx.contentResolver.openFileDescriptor(uri, "r")!!.use { pfd ->
        FileInputStream(pfd.fileDescriptor).channel.use { ch ->
            val buf = ByteBuffer.allocate((to - from).toInt())
            ch.read(buf, from); return buf.array()
        }
    }
}

class WebSocketSession(val http: OkHttpClient, val url: String) {
    fun incoming(): Flow<Frame> = callbackFlow {
        var backoff = 1000L
        while (isActive) {
            val req = Request.Builder().url(url).build()
            val ws = http.newWebSocket(req, object : WebSocketListener() {
                override fun onOpen(ws: WebSocket, r: Response) { backoff = 1000L }
                override fun onMessage(ws: WebSocket, t: String) { trySend(parse(t)) }
                override fun onFailure(ws: WebSocket, t: Throwable, r: Response?) { close() }
                override fun onClosed(ws: WebSocket, c: Int, r: String) { close() }
            })
            awaitClose { ws.close(1000, null) }
            delay(backoff); backoff = (backoff * 2).coerceAtMost(60_000)
        }
    }
}

class BackupScanner(val ctx: Context, val dao: LocalAssetDao) {
    fun start() {
        ctx.contentResolver.registerContentObserver(
            MediaStore.Images.Media.EXTERNAL_CONTENT_URI, true,
            object : ContentObserver(null) {
                override fun onChange(self: Boolean) { scope.launch { discover() } }
            })
    }

    private suspend fun discover() {
        val known = dao.allMediaIds().toHashSet()
        ctx.contentResolver.query(
            MediaStore.Files.getContentUri("external"),
            arrayOf(MediaStore.Files.FileColumns._ID, MediaStore.Files.FileColumns.SIZE),
            null, null, null
        )?.use { c ->
            while (c.moveToNext()) {
                val id = c.getLong(0); if (id in known) continue
                val uri = ContentUris.withAppendedId(
                    MediaStore.Files.getContentUri("external"), id)
                val hash = sha256(ctx, uri)
                dao.insert(LocalAsset(id, uri.toString(), hash))
            }
        }
        scheduleBackup()
    }

    private fun scheduleBackup() {
        WorkManager.getInstance(ctx).enqueueUniqueWork(
            "backup", ExistingWorkPolicy.KEEP,
            OneTimeWorkRequestBuilder<BackupWorker>()
                .setConstraints(Constraints.Builder()
                    .setRequiredNetworkType(NetworkType.UNMETERED)
                    .setRequiresBatteryNotLow(true)
                    .setRequiresStorageNotLow(true)
                    .build())
                .build()
        )
    }
}

class BackupWorker(c: Context, p: WorkerParameters) : CoroutineWorker(c, p) {
    override suspend fun doWork(): Result {
        val next = dao.nextPending() ?: return Result.success()
        if (api.exists(next.hash)) {                // dedupe
            dao.markUploaded(next.mediaId, api.refOf(next.hash)); return Result.retry()
        }
        val remoteId = uploadResumable(next)        // see #15
        dao.markUploaded(next.mediaId, remoteId)
        return Result.retry()                        // process more
    }
}

class ExoPlayerPool(ctx: Context, size: Int = 3) {
    private val available = ArrayDeque<ExoPlayer>().apply {
        repeat(size) { add(ExoPlayer.Builder(ctx).build()) }
    }
    private val assignments = mutableMapOf<String, ExoPlayer>()

    fun bind(itemId: String, uri: String): ExoPlayer {
        assignments[itemId]?.let { return it }
        val p = available.removeFirstOrNull() ?: evictFarthest()
        p.setMediaItem(MediaItem.fromUri(uri)); p.prepare()
        assignments[itemId] = p; return p
    }
    fun release(itemId: String) {
        assignments.remove(itemId)?.let { it.stop(); available.addLast(it) }
    }
    fun dispose() { (available + assignments.values).forEach { it.release() } }
}

class LruCache<K, V>(private val cap: Int) {
    private class Node<K, V>(val key: K, var value: V) {
        var prev: Node<K, V>? = null
        var next: Node<K, V>? = null
    }
    private val map = HashMap<K, Node<K, V>>()
    private val head = Node<K, V>(null as K, null as V)
    private val tail = Node<K, V>(null as K, null as V)
    init { head.next = tail; tail.prev = head }

    fun get(key: K): V? = map[key]?.also { moveToHead(it) }?.value

    fun put(key: K, value: V) {
        map[key]?.let { it.value = value; moveToHead(it); return }
        val node = Node(key, value)
        map[key] = node; addToHead(node)
        if (map.size > cap) {
            val evict = tail.prev!!
            remove(evict); map.remove(evict.key)
        }
    }

    private fun addToHead(n: Node<K, V>) {
        n.prev = head; n.next = head.next; head.next!!.prev = n; head.next = n
    }
    private fun remove(n: Node<K, V>) {
        n.prev!!.next = n.next; n.next!!.prev = n.prev
    }
    private fun moveToHead(n: Node<K, V>) { remove(n); addToHead(n) }
}

class TokenBucket(val capacity: Long, val refillPerSec: Double) {
    private var tokens = capacity.toDouble()
    private var last = System.nanoTime()

    @Synchronized
    fun tryAcquire(n: Long = 1): Boolean {
        refill()
        return if (tokens >= n) { tokens -= n; true } else false
    }

    private fun refill() {
        val now = System.nanoTime()
        val elapsedSec = (now - last) / 1e9
        tokens = (tokens + elapsedSec * refillPerSec).coerceAtMost(capacity.toDouble())
        last = now
    }
}

class RateLimitInterceptor(val bucket: TokenBucket) : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        while (!bucket.tryAcquire()) Thread.sleep(25)   // or throw 429-style
        return chain.proceed(chain.request())
    }
}

enum class Direction { UP, DOWN, IDLE }

class Elevator(val id: Int, var floor: Int, val minF: Int, val maxF: Int) {
    val upQueue = sortedSetOf<Int>()
    val downQueue = sortedSetOf<Int>(reverseOrder())
    var direction = Direction.IDLE

    fun add(target: Int) {
        if (target > floor) upQueue.add(target) else if (target < floor) downQueue.add(target)
    }

    fun tick(): Int? {
        if (direction == Direction.UP || (direction == Direction.IDLE && upQueue.isNotEmpty())) {
            direction = Direction.UP
            if (upQueue.isEmpty()) { direction = Direction.DOWN; return tick() }
            floor++
            if (floor in upQueue) { upQueue.remove(floor); return floor }
        } else if (direction == Direction.DOWN) {
            if (downQueue.isEmpty()) { direction = Direction.IDLE; return null }
            floor--
            if (floor in downQueue) { downQueue.remove(floor); return floor }
        }
        return null
    }

    fun score(call: HallCall): Int {
        val dist = abs(call.floor - floor)
        val mismatch = if (direction != Direction.IDLE && direction != call.dir) 100 else 0
        return dist + mismatch
    }
}

class Scheduler(val elevators: List<Elevator>) {
    fun handle(call: HallCall) {
        val chosen = elevators.minByOrNull { it.score(call) } ?: return
        chosen.add(call.floor)
    }
}

    fun onBoard() = r in 0..7 && c in 0..7
}

sealed class Piece(val color: Color) {
    abstract fun pseudoMoves(board: Board, from: Square): List<Move>
}

class Knight(c: Color) : Piece(c) {
    override fun pseudoMoves(b: Board, f: Square) =
        listOf(-2 to -1, -2 to 1, -1 to -2, -1 to 2,
               1 to -2, 1 to 2, 2 to -1, 2 to 1)
            .map { Square(f.r + it.first, f.c + it.second) }
            .filter { it.onBoard() && b.at(it)?.color != color }
            .map { Move(f, it) }
}

class Game(val board: Board) {
    var turn: Color = Color.WHITE
    val history = mutableListOf<Move>()
    val castlingRights = mutableMapOf<Color, MutableSet<CastleSide>>()
    var enPassant: Square? = null

    fun legalMoves(from: Square): List<Move> {
        val p = board.at(from) ?: return emptyList()
        if (p.color != turn) return emptyList()
        return p.pseudoMoves(board, from).filter { mv ->
            board.simulate(mv) { !board.isInCheck(turn) }
        }
    }

    fun play(move: Move): Boolean {
        if (move !in legalMoves(move.from)) return false
        board.apply(move); history += move
        turn = if (turn == Color.WHITE) Color.BLACK else Color.WHITE
        return true
    }
}

enum class Mark { X, O; fun opp() = if (this == X) O else X }

class Board(val n: Int, val k: Int) {
    private val g = Array(n) { arrayOfNulls<Mark>(n) }
    fun place(r: Int, c: Int, m: Mark): Boolean {
        if (g[r][c] != null) return false
        g[r][c] = m; return true
    }
    fun at(r: Int, c: Int) = g[r][c]
    fun winsAt(r: Int, c: Int): Boolean {
        val m = g[r][c] ?: return false
        val dirs = listOf(0 to 1, 1 to 0, 1 to 1, 1 to -1)
        return dirs.any { (dr, dc) -> runLength(r, c, dr, dc, m) >= k }
    }
    private fun runLength(r: Int, c: Int, dr: Int, dc: Int, m: Mark): Int {
        var count = 1
        var rr = r + dr; var cc = c + dc
        while (rr in 0 until n && cc in 0 until n && g[rr][cc] == m)
            { count++; rr += dr; cc += dc }
        rr = r - dr; cc = c - dc
        while (rr in 0 until n && cc in 0 until n && g[rr][cc] == m)
            { count++; rr -= dr; cc -= dc }
        return count
    }
}

fun minimax(b: Board, last: Triple<Int,Int,Mark>?, toMove: Mark, depth: Int = 0): Int {
    last?.let { if (b.winsAt(it.first, it.second)) return if (it.third == toMove) -10 + depth else 10 - depth }
    var best = -100; var any = false
    for (r in 0 until b.n) for (c in 0 until b.n) {
        if (b.at(r, c) == null) {
            any = true; b.place(r, c, toMove)
            best = maxOf(best, -minimax(b, Triple(r, c, toMove), toMove.opp(), depth + 1))
            b.undo(r, c)
        }
    }
    return if (any) best else 0
}

class CashDispenser(private val inventory: MutableMap<Int, Int>) {
    fun dispense(amount: Int): Map<Int, Int>? {
        val dens = inventory.keys.sortedDescending()
        val plan = mutableMapOf<Int, Int>()
        var rem = amount
        for (d in dens) {
            val take = minOf(rem / d, inventory.getValue(d))
            if (take > 0) { plan[d] = take; rem -= take * d }
            if (rem == 0) break
        }
        if (rem != 0) return null
        plan.forEach { (d, n) -> inventory[d] = inventory.getValue(d) - n }
        return plan
    }
}

sealed class AtmState {
    object Idle : AtmState()
    data class AwaitingPin(val card: Card) : AtmState()
    data class Authenticated(val account: Account) : AtmState()
}

class Atm(val bank: BankApi, val cash: CashDispenser) {
    var state: AtmState = AtmState.Idle; private set

    fun insert(card: Card) { state = AtmState.AwaitingPin(card) }

    suspend fun enterPin(pin: String) {
        val s = state as? AtmState.AwaitingPin ?: return
        val account = bank.auth(s.card, pin) ?: run { eject(); return }
        state = AtmState.Authenticated(account)
    }

    suspend fun withdraw(amount: Int): WithdrawResult {
        val s = state as? AtmState.Authenticated ?: return WithdrawResult.NotAuth
        if (s.account.balance < amount) return WithdrawResult.Insufficient
        val plan = cash.dispense(amount) ?: return WithdrawResult.CannotMakeChange
        val ok = bank.debit(s.account, amount)
        if (!ok) { rollback(plan); return WithdrawResult.BankRefused }
        return WithdrawResult.Ok(plan)
    }
}

class VendingMachine(
    val products: MutableMap<String, Product>,
    val cashBox: MutableMap<Int, Int>
) {
    private var state: State = State.Idle
    private var deposit = 0
    private var selectedSlot: String? = null

    fun select(slot: String): SelectResult {
        val p = products[slot] ?: return SelectResult.Invalid
        if (p.qty == 0) return SelectResult.Sold
        selectedSlot = slot; state = State.AwaitingPayment
        return SelectResult.Need(p.priceCents)
    }

    fun insert(coin: Int): InsertResult {
        if (state != State.AwaitingPayment) return InsertResult.NotAccepting
        cashBox[coin] = (cashBox[coin] ?: 0) + 1
        deposit += coin
        val p = products.getValue(selectedSlot!!)
        return if (deposit >= p.priceCents) {
            val change = deposit - p.priceCents
            val plan = makeChange(change) ?: run { refund(); return InsertResult.NoChange }
            p.qty -= 1; deposit = 0; selectedSlot = null; state = State.Idle
            InsertResult.Dispense(p.name, plan)
        } else InsertResult.NeedMore(p.priceCents - deposit)

35. Splitwise

30-sec elevator pitch: "Maintain net per user (+ receives, − pays). To simplify: sort into max-heap of givers (most negative) and max-heap of receivers (most positive); repeatedly settle the largest pair until empty. Greedy is O(N log N) and optimal in practice (true optimum is NP-hard)."

data class Expense(val payer: String, val amount: Long,
                   val splits: Map<String, Long>)         // userId -> share in cents

class Splitwise {
    private val netByUser = HashMap<String, Long>()       // + receives, - pays

    fun add(e: Expense) {
        netByUser.merge(e.payer, e.amount) { a, b -> a + b }
        e.splits.forEach { (u, s) ->
            netByUser.merge(u, -s) { a, b -> a + b }
        }
    }

    fun simplify(): List<Settlement> {
        val givers = TreeSet<Pair<Long, String>>(compareBy { it.first })   // most negative first
        val receivers = TreeSet<Pair<Long, String>>(compareBy { -it.first })// most positive first
        netByUser.forEach { (u, n) ->
            when { n < 0 -> givers += n to u; n > 0 -> receivers += n to u }
        }
        val out = mutableListOf<Settlement>()
        while (givers.isNotEmpty() && receivers.isNotEmpty()) {
            val g = givers.first(); val r = receivers.first()
            val pay = minOf(-g.first, r.first)
            out += Settlement(from = g.second, to = r.second, amount = pay)
            givers.remove(g); receivers.remove(r)
            if (-g.first > pay) givers += (g.first + pay) to g.second
            if (r.first  > pay) receivers += (r.first - pay) to r.second
        }
        return out
    }
}

                enc.releaseOutputBuffer(idx, false)

data class FlagConfig(
    val key: String,
    val default: String,
    val variants: List<String>,
    val allocation: List<Int>,         // must sum to 100
    val minAppVersion: Int = 0
)

object Flags {
    @Volatile private var configs: Map<String, FlagConfig> = emptyMap()
    private val exposed = ConcurrentHashMap.newKeySet<String>()
    private lateinit var userId: String
    private lateinit var session: String
    private lateinit var appVersion: () -> Int

    fun init(uid: String, sid: String, ver: () -> Int, dao: FlagDao) {
        userId = uid; session = sid; appVersion = ver
        configs = dao.all().associateBy { it.key }
        scope.launch { refresh(dao) }
    }

    fun value(key: String, default: String): String {
        val cfg = configs[key] ?: return default
        if (appVersion() < cfg.minAppVersion) return default
        val b = bucket(userId, key)
        var acc = 0
        cfg.allocation.forEachIndexed { i, pct ->
            acc += pct; if (b < acc) {
                logExposure(key, cfg.variants[i])
                return cfg.variants[i]
            }
        }
        return default
    }

    private fun bucket(uid: String, key: String): Int {
        val bytes = MessageDigest.getInstance("MD5").digest("$uid:$key".toByteArray())
        val n = ByteBuffer.wrap(bytes, 0, 4).int and Int.MAX_VALUE
        return n % 100
    }

    private fun logExposure(key: String, variant: String) {
        val dedupeKey = "$session|$key"
        if (exposed.add(dedupeKey))
            Analytics.track("exposure", mapOf("key" to key, "variant" to variant))
    }

    private suspend fun refresh(dao: FlagDao) {
        runCatching {
            val resp = api.flags(etag = dao.etag())
            if (resp.changed) { dao.replaceAll(resp.flags); configs = resp.flags.associateBy { it.key } }
        }
    }
}

class CrashHandler(val ctx: Context) : Thread.UncaughtExceptionHandler {
    private val previous = Thread.getDefaultUncaughtExceptionHandler()
    private val dir = File(ctx.filesDir, "crashes").apply { mkdirs() }

    fun install() { Thread.setDefaultUncaughtExceptionHandler(this) }

    override fun uncaughtException(t: Thread, e: Throwable) {
        try {
            val report = CrashReport(
                id = UUID.randomUUID().toString(),
                threadName = t.name,
                stack = e.stackTraceToString(),
                breadcrumbs = Breadcrumbs.snapshot(),
                deviceModel = Build.MODEL,
                osVersion = Build.VERSION.SDK_INT,
                appVersion = BuildConfig.VERSION_NAME,
                memoryUsed = Debug.getNativeHeapAllocatedSize(),
                ts = System.currentTimeMillis()
            )
            File(dir, "${report.id}.crash").writeText(Json.encodeToString(report))
        } catch (_: Throwable) { /* swallow — never throw inside handler */ }
        finally { previous?.uncaughtException(t, e) }       // chain
    }
}

class ANRWatcher(val ctx: Context, val thresholdMs: Long = 5000) {
    private val main = Handler(Looper.getMainLooper())
    private val exec = Executors.newSingleThreadScheduledExecutor()

    fun start() {
        exec.scheduleAtFixedRate({
            val ack = AtomicBoolean(false)
            main.post { ack.set(true) }
            Thread.sleep(thresholdMs)
            if (!ack.get()) captureANR()
        }, 5, 5, TimeUnit.SECONDS)
    }

    private fun captureANR() {
        val stacks = Thread.getAllStackTraces()
            .entries.joinToString("\n\n") { (t, s) ->
                "Thread ${t.name}\n" + s.joinToString("\n") { "  at $it" }
            }
        File(ctx.filesDir, "crashes/anr-${System.currentTimeMillis()}.crash")
            .writeText(stacks)
    }
}

    val api: CheckoutApi,
    val payment: PaymentSdk
) {

        val items = dao.all()
        val latest = api.priceCheck(items.map { it.sku })
        val changes = items.mapNotNull { local ->
            val remote = latest[local.sku] ?: return@mapNotNull null
            if (remote != local.snapshotPriceCents)

    }

// Phone-side: write to DataLayer
class ProfileSyncer(val ctx: Context) {
    private val dataClient = Wearable.getDataClient(ctx)

    suspend fun push(profile: Profile) {
        val req = PutDataMapRequest.create("/profile").apply {
            dataMap.putString("name", profile.name)
            dataMap.putInt("goal", profile.dailyStepGoal)
            dataMap.putLong("updatedAt", profile.updatedAt)
        }.asPutDataRequest().setUrgent()
        dataClient.putDataItem(req).await()
    }
}

// Watch-side: listen for changes
class ProfileListenerService : WearableListenerService() {
    override fun onDataChanged(events: DataEventBuffer) {
        events.filter { it.type == DataEvent.TYPE_CHANGED && it.dataItem.uri.path == "/profile" }
            .forEach { e ->
                val map = DataMapItem.fromDataItem(e.dataItem).dataMap
                profileDao.upsert(Profile(
                    name = map.getString("name")!!,
                    dailyStepGoal = map.getInt("goal"),
                    updatedAt = map.getLong("updatedAt")
                ))
            }
    }
}

6 Cross-questions & answers
Q. Phone is in another room — does the watch still work?
A. Yes, if marked standalone in the manifest (com.google.android.wearable.standalone = true) and we have a direct internet path on the watch (LTE / Wi-Fi). Sync queue drains when phone reconnects.
Q. Live workout heart-rate — DataLayer or Channel?
A. ChannelClient — DataLayer is for state, not high-frequency streams. Health Services API for the actual sensor read on the watch.
Q. Battery during workout?
A. Use Health Services (batched samples, hardware-accelerated) instead of raw Sensors API; FG service with type health; reduce screen brightness and tick rate.
Q. Tile updates — how often?
A. Tiles refresh on a schedule (max once / 20 min in background per platform constraints). Use TileService.requestUpdate for explicit refreshes from a worker.
Q. Notifications mirror or independent?
A. By default Android mirrors phone notifications to watch. For watch-only experiences (Tile actions), build the notification on the watch via NotificationManager there.
7 What can be better / alternatives
  • Health Connect for unified health-data sharing across apps.
  • Watch Face Push API for first-party watch face customization.
  • Material 3 for Wear components for ergonomic round-screen UI.
  • Exposure to FCM directly on watch for standalone notifications.
Don't say: "I'll show the same UI as the phone, scaled down." Round-screen UX, glanceability, and battery constraints demand watch-first design (Tiles, Complications).
